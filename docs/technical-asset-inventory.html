1. Delta Live Tables (DLT) Pipelines

The platform uses a three-tier medallion architecture implemented with Delta Live Tables for declarative, scalable ETL processing.

DLT Pipeline Components

Component File Path Layer Description
Bronze Pipeline core/dlt/bronze.py Bronze Raw telemetry ingestion from Zerobus and PI simulator. Normalizes OT/IoT streams into canonical schema. Applies data quality constraints.
Silver Pipeline core/dlt/silver.py Silver Feature engineering with 15m/1h/30d windows. OT+PI time-alignment with provenance tracking. Quality-weighted aggregations.
Gold Pipeline core/dlt/gold.py Gold ML-ready feature vectors. Prediction enrichment with financial context. Maintenance event materialization with ROI calculations.

Key DLT Functions

Bronze: Sensor Readings

Function: @dlt.table sensor_readings()

Streams from Zerobus landing table. Validates ISA-95 hierarchy (site/area/unit/equipment). Enforces quality codes (GOOD, UNCERTAIN, BAD).

Bronze: PI Simulated Tags

Function: @dlt.table pi_simulated_tags()

Ingests PI historian simulation data. Mirrors OT schema for alignment. Preserves source protocol metadata.

Silver: Sensor Features

Function: @dlt.table sensor_features()

Computes mean, stddev, slope, z-score over multiple windows. Quality-weighted aggregations. Excludes BAD quality readings.

Silver: OT-PI Alignment

Function: @dlt.table ot_pi_aligned()

Time-proximity join (30s window). Provenance tracking: BOTH, OT_ONLY, PI_ONLY, UNKNOWN. Delta computation for drift analysis.

Gold: PDM Predictions

Function: @dlt.table pdm_predictions()

Enriches ML model outputs with asset context. Links anomaly scores to RUL estimates. Identifies top contributing sensors.

Gold: Financial Impact

Function: @dlt.table financial_impact_events()

Merges predictions with maintenance windows, crew availability, parts inventory. Computes expected failure cost, avoided cost, total event cost.

3. Machine Learning Components

MLflow-managed model lifecycle with per-asset anomaly detection (Isolation Forest) and remaining useful life (RUL) regression models.

ML Training & Scoring Modules

Module File Path Purpose Key Functions
Anomaly Model core/ml/anomaly_model.py Isolation Forest for unsupervised anomaly detection OTPdMAnomalyModel.fit(), score(), log_to_mlflow()
RUL Model core/ml/rul_model.py Remaining useful life regression (Random Forest) OTPdMRULModel.fit(), predict(), evaluate()
Training Job core/ml/train.py Per-industry model training orchestration train_asset_models(), MLflow experiment tracking
Batch Scoring core/ml/batch_score.py Batch prediction pipeline (scheduled every 2 hours) score_asset(), champion model resolution, output to Gold
Feature Engineering core/ml/features.py Feature matrix construction from Silver tables get_feature_matrix(), pivoting, time-window aggregation
Feature Importance core/ml/feature_importance.py SHAP and permutation importance analysis compute_feature_importance(), sensor contribution ranking
Model Evaluation core/ml/evaluate.py RMSE, R², MAE metrics for RUL models evaluate_rul_model(), precision/recall for anomaly

Anomaly Detection

Algorithm: Isolation Forest (scikit-learn)

Training data: Healthy operation window (first N hours before fault injection)

Output: Anomaly score (0-1), binary label (NORMAL/ANOMALY), top contributing sensor

RUL Prediction

Algorithm: Random Forest Regressor

Label generation: Hours to failure from fault_start_offset

Output: RUL in hours, predicted failure timestamp, confidence interval

MLflow Tracking

Experiment: /Shared/ot-pdm-intelligence/{industry}

Registered models: pdm_{industry}_{equipment_id}_anomaly, _rul

Model registry: Unity Catalog (databricks-uc), Champion/Challenger aliasing

Currency & Localization Logic

Backend implements multi-currency conversion with site-native defaults:

  • Supported currencies: USD, AUD, JPY, INR, SGD, KRW, AUTO (site-native)
  • All financial calculations done in base currency, converted on response
  • Language behavior: JPY → Japanese text, KRW → Korean text, others → English
  • Geo APIs accept ?currency=XXX parameter for dynamic conversion
